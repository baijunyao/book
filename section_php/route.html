Laravel-Route

Router (路由)

laravel&lumen路由设计非常优雅，非常具有借鉴意义。

下面讲的主要是laravel的实现方式。

laravel 把路由模型分成(Router,Route, RouteCollection)三个主要模型，我认为这是领域驱动设计经典的案例。

这三个组件的职责： Router: 注册和匹配路由，对外暴露所有操作路由的接口 Route: 路由的基础类，定义路由模型所涉及的属性和对应方法 RouteCollection: 路由的集合

基本的逻辑是: 1 注册路由：Router把Route注册到RouteCollection 2 路由请求：Router通过Request信息在RouteCollection中查找到对应的Route, 并执行Route的run方法返回response

Route

路由定义是【负责应答URI的请求】，把应答的方式抽象为Action，那么路由实际上就是由URI和与之对应的Action两部分组成。

URI好理解，是统一资源标识符 例如:

GET /v1/users/1
GET /v2/users/1/comments/2
GET /v3/search/

Action是什么意思呢

Action就是当匹配到路由后执行的动作，在laravel中，Action定义为一个数组，经过parse后，最终生成的的action的格式如下:

# action是控制器的某个方法
action => Array(
    as => profile,  # 路由的别名
    uses => ExampleController@index,
)

# action 是匿名函数
action => Array(
    as => profile,  # 路由的别名
    uses => Closure Object(),  # 执行一个闭包函数, 这里留一个问题，闭包函数支持传递哪些参数
)

如果把URL和Action组合在一起，一个路由的结构是这样

# action是执行控制器的某个方法
route = Array (
    [method] => GET  # 定义的请求方法
    [uri] => /v1/profiles/{id: \d+} # 定义的URI
    [action] => Array
        (
            [as] => profile
            [uses] => App\Http\Controllers\ExampleController@index
        )
)
# action是执行闭包函数
route => Array
(
    [method] => GET
    [uri] => /v1/users/{user_id}
    [action] => Array (
            [as] => profile
            [uses] => Closure Object (
                [parameter] => Array
                (
                    [$user_id] => <required>
                )
            )
        )
)

method, uri, action是Route的核心概念

Route所有的操作就是围绕这三个基础属性展开的。初始化一个Route的时候需要传递这三个元素:

public function __construct($methods, $uri, $action)
{
    $this->uri = $uri;
    $this->methods = (array) $methods;
    $this->action = $this->parseAction($action);

    if (in_array('GET', $this->methods) && ! in_array('HEAD', $this->methods)) {
        $this->methods[] = 'HEAD';
    }

    if (isset($this->action['prefix'])) {
        $this->prefix($this->action['prefix']);
    }
}

Method

Method 是对应的HTTP Request的method，目前支持GET POST PUT DELETE HEAD等，这里需要特别注意，一个URI可以对应多个Method.

URI

URI是路由的重点也是难点，

URI有prefix(前缀的概念) 可以把版本信息加入到路由的前缀中, 在Route类里面有个方法prefix可以为uri添加前缀

public function prefix($prefix)
{
    $uri = rtrim($prefix, '/').'/'.ltrim($this->uri, '/');

    $this->uri = trim($uri, '/');

    return $this;
}

定义URI时可以加入变量, 下面的例子中，uri加入了user_id, comment_id这两个变量

# 匹配的url: /v2/users/1/comments/1
/v2/users/{user_id}/comments/{comment_id}

在laravel 中，还可以给这些变量定义格式, 使用Router中的pattern方法:

$router = new Router()
# user_id必须是数字
$router->pattern("user_id", `\d+`);
# comment_id 必须是数字
$router->pattern("comment_id", `\d+`);

在lumen中，这个规则可以直接定义在uri里面:

/users/{user_id: \d+}
/v2/users/{user_id: \d+}/comments/{comment_id: \d+}

在laravel 中 Route会uri里面的变量以及从变量中匹配到的值保存在parameters 这样一个数组中.

解析一个uri完成后，parameters 可能为:

 $parameters = [
    "user_id": 1,
    "comment_id": 3
]

那parameters 是什么时候被用到的呢，先留这个问题在这里。

最后一个问题，uri是怎么匹配的呢， uri匹配我们最关心的是，匹配url里面的变量

laravel的做法是，首先complie的路由，生成一套规则，然后根据reqeust的pathinfo 信息匹配到对应的路由，其中uri的变量匹配有个一比较重要代码


$pattern = $uri # 定义的uri
preg_match_all('#\{\w+\}#', $pattern, $matches, PREG_OFFSET_CAPTURE | PREG_SET_ORDER);

具体的算法可自行查看代码。

URI匹配成功后，就到了执行URI对应的action了

Action

上文提到 Action有两种方式，执行一个控制器的方法 格式为controller@method, 还有一个是闭包函数。

先讨论action为闭包函数

# 比如定义这样的一个路由
App::get('/uses/{id}', function($id) {
    return $id;
});

# 或者这样定义
App::get('/uses/{id}', array('as'=> 'users', function($id) {
    return $id;
}));

# 这个路由被parse之后，生成的action
action = array(
    'as' => 'users',
    'uses' => Closure Object(
        "parameters" => $id,
    )
);

执行action 就是执行这个闭包函数，有个问题是闭包函数的参数是哪里赋值的？

还记得上文提到的Route里面有一个parameters数组么， 闭包函数里面的参数就是parameters的值。

大概的代码是这样:


protected function runCallable()
{
    $callable = $this->action['uses'];

    # parametersWithoutNulls 就是从parameters过滤了null的值
    # 通过反射闭包函数，找到闭包函数的参数，然后从parametersWithoutNulls中拿去对应的值
    return $callable(...array_values($this->resolveMethodDependencies(
        $this->parametersWithoutNulls(), new ReflectionFunction($this->action['uses'])
    )));
}

现在讨论action是Controller@method的

 # 假设定义这样的路由
 App::get('/profiles/{id: \d+}', array("as" => "profile", "uses" => "ExampleController@index", ));

 # 生成的action是
 action = array(
    'as' => 'profile',
    'uses' => "ExampleController@index",
 );

执行这类action的逻辑是这样的，找到controller并实例化controller执行controller对应的mehtod。

和闭包函数的参数一样，controller中method的参数也通过反射自动从parametersWithoutNulls 获取。

protected function runController()
{
    return (new ControllerDispatcher($this->container))->dispatch(
        $this, $this->getController(), $this->getControllerMethod()
    );
}

最后还有一个知识点，是action的as 别名怎么使用。

as是为了支持路由重定向，类似于nginx的rewrite 是程序内部的跳转，不会修改到uri。

但uri定制了别名后，RouteCollection里面有个数组nameList 会保存别名到action的映射关系，那通过别名就可以直接找到action了,

laravel 提供了一个route函数可以对路由转发

比如这样:

$nameList['profile'] = 'ExampleController@index'

# 执行route对应的action
route('profile');
